The Mission

You're about to go through ALL of these steps:

  1. Create a repository on GitHub
  2. Clone it to your computer
  3. Add your Scratch project file
  4. Push it back to GitHub
  5. Submit the link on Canvas

Let's do this.

Phase 1: Create Repository

Make a new repo on GitHub.com

Go to GitHub

  1. Open github.com in your browser
  2. Sign in to your account
  3. Click the + button in the top right corner
  4. Select New repository

Phase 1: Create Repository

Make a new repo on GitHub.com

Set Up Your Repository

  1. Repository name: Enter lmsc261
  2. Description: (optional) Something like "My class projects"
  3. Public or Private: Choose Private
  4. Initialize with a README: Check this box
  5. Click Create repository

Phase 1: Create Repository

Make a new repo on GitHub.com

Add Your Instructor as a Collaborator

  1. On your new repository page, click Settings (gear icon)
  2. In the left sidebar, click Collaborators
  3. Click Add people
  4. Enter your instructor's GitHub username: Sam-Zilli
  5. Click Add Sam-Zilli to this repository
Important: If you skip this step, your instructor won't be able to see your submissions.

Phase 2: Clone to Computer

Download the repo to your computer

Copy the Repository URL

  1. Go back to your repository's main page (click the repo name at the top)
  2. Click the green Code button
  3. Make sure HTTPS is selected
  4. Click the copy icon to copy the URL

The URL will look like: https://github.com/yourusername/lmsc261.git

Phase 2: Clone to Computer

Download the repo to your computer

Clone in VS Code

  1. Open VS Code
  2. Press Cmd + Shift + P to open the Command Palette
  3. Type Git: Clone and select it
  4. Paste the URL you copied
  5. Choose where to save it (like your Desktop or Documents folder)
  6. When asked, click Open to open the cloned repository
What just happened? You downloaded a copy of your GitHub repository to your computer. Now you can add files to it.

Phase 3: Add Your Project

Get your Scratch file into the repo

Download Your Scratch Project

  1. Go to scratch.mit.edu
  2. Open your project
  3. Click File in the menu bar
  4. Click Save to your computer
  5. Save the .sb3 file somewhere you can find it
Tip: The file will be named something like my-project.sb3

Phase 3: Add Your Project

Get your Scratch file into the repo

Add the File to Your Repository

  1. Find your downloaded .sb3 file in Finder
  2. Drag it into your VS Code window (into the file explorer on the left)
  3. You should see your file in VS Code (but it won't be able to read it) like this:
Note: VS Code can't open .sb3 files because they're Scratch project files, not text files. This is totally normal! You just need the file to exist in your repository.

Phase 4: Push to GitHub

Upload your changes

Commit and Push

  1. In VS Code, click the Source Control icon in the left sidebar (or press Ctrl + Shift + G)
  2. You'll see your .sb3 file listed under "Changes"
  3. Click the + next to the file to stage it
  4. Type a commit message like "Add scratch project"
  5. Click the checkmark to commit
  6. Click Sync Changes (or the cloud icon) to push to GitHub
First time? VS Code might ask you to sign in to GitHub. Follow the prompts to authorize VS Code.

Phase 5: Verify on GitHub

Make sure your file is there

Verify on GitHub

  1. Open a web browser and go to github.com
  2. Sign in if you're not already
  3. Click on your lmsc261 repository (you should see it in your repositories list)
  4. Look at the file list on the repository page
  5. You should see your .sb3 file listed there along with your README.md

Check:

Your .sb3 file is visible on GitHub
Sam-Zilli is added as a collaborator

Phase 6: Submit on Canvas

Turn in your work

Submit on Canvas

  1. Copy your repository URL from the browser (e.g., https://github.com/yourusername/lmsc261)
  2. Go to the assignment on Canvas
  3. Paste the URL and submit

You Did It!

You've completed the full developer workflow.

Create → Clone → Add → Commit → Push → Submit
