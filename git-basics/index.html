The Git Workflow

When you make changes to your code, you need to save them to GitHub. This happens in three steps:

Why three steps?

  • Add - Choose which changes you want to save
  • Commit - Save those changes with a message describing what you did
  • Push - Upload your commits to GitHub so others can see them

Step 1: Make a Change

First, let's make a change to your repository.

  1. Open your Intro-To-Programming folder in VS Code
  2. Open the README.md file
  3. Add a new line with some text, like:
    This is my first edit!
  4. Save the file (Cmd + S)
Notice: After saving, look at the Source Control icon in the left sidebar. It should show a number (1) indicating you have one changed file.

Step 2: Add (Stage Your Changes)

"Adding" tells Git which changes you want to include in your next save.

In VS Code:

  1. Click the Source Control icon in the left sidebar (looks like a branch)
  2. You'll see your changed file listed under "Changes"
  3. Hover over the file and click the + button to stage it
What happened? The file moved from "Changes" to "Staged Changes". This means it's ready to be committed.

Terminal Alternative:

git add README.md

Or to add all changed files:

git add .

Step 3: Commit (Save Your Changes)

A "commit" is like a save point. It records your changes with a message describing what you did.

In VS Code:

  1. In the Source Control panel, find the message box at the top
  2. Type a short message describing your change, like:
    Add first edit to README
  3. Click the Commit button (or press Cmd + Enter)
Good commit messages: Be specific about what you changed. "Updated README" is okay, but "Add project description to README" is better.

Terminal Alternative:

git commit -m "Add first edit to README"

Step 4: Push (Upload to GitHub)

Your changes are saved locally, but GitHub doesn't know about them yet. "Push" uploads your commits.

In VS Code:

  1. After committing, click the Sync Changes button (or the cloud icon with an arrow)
  2. If prompted, click "OK" to push and pull
Check it worked: Go to your repository on GitHub.com and refresh the page. You should see your changes in the README file.

Terminal Alternative:

git push

Quick Reference

Here's the workflow you'll use every time you want to save changes:

VS Code

  1. Save your file
  2. Click + to stage
  3. Type a message
  4. Click Commit
  5. Click Sync Changes

Terminal

  1. Save your file
  2. git add .
  3. git commit -m "message"
  4. git push
