Follow the steps below to add a local file to your GitHub repository and push it to your remote repository.

Step 1: Scratch File into Local Repo

Using the commands we have learned in class, move your Scratch file (ends in .sb3) into the repository you cloned onto your computer.

Step 2: Add Your File to the Repository

Even though your file is in the repo's directory, Git does not yet KNOW about your file.

Add your file to Git's tracking by running: 

git add <your-scratch-project.sb3>

This stages your file to be committed.

Step 3: Commit the Changes

Now that your file is staged, you need to commit it. A commit is like a snapshot of your changes. Run the following command:

git commit -m "Added scratch project to repo"

Step 4: Push Your Changes to GitHub

Once you've committed your changes, push them to your remote GitHub repository using the following command:

git push

NOTE: If it asks for your username, enter it. If it asks for your password ENTER THE AUTH TOKEN you generated on the last page - NOT YOUR GITHUB PASSWORD

Explanation:

  • git push - pushes your committed changes to the remote repository.

After running this command, your local file will be uploaded to your GitHub repository.

Step 5: Verify Your Changes on GitHub.com

Follow these steps to verify your changes:

  • Go to GitHub.com.
  • Click on the three lines in the top-left corner to open the sidebar.
  • In the pane that opens, scroll down to "Repositories" and find the repository for this project.
  • Click on the repository to open it.
  • Check if your scratch file and the README.md file are both listed there.

If everything went smoothly, congratulations! You've successfully added a local file to your GitHub repository and pushed it to the remote.

This is how you will submit coding assignments from now on!
