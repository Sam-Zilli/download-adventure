How to Create an Auth Token on GitHub.com

  1. Step 1: Go to Your Settings

    Login to GitHub.com. Click on your profile picture in the top right corner, and select Settings from the dropdown.

  2. Step 2: Access Developer Settings

    In the left sidebar, scroll down and click on Developer settings (located at the very bottom).

  3. Step 3: Click on "Personal access tokens"

    In the Developer settings menu, select Personal access tokens - then Tokens (classic).

  4. Step 4: Create a New Token

    Click the Generate new token button.

    Note: Choose Classic. You may be asked to enter your GitHub password to continue.

  5. Step 5: Set Name and Permissions for Your Token

    Give your token a name such as lmsc261. Set the expiration for 90 days.

    Below the expiration date, you'll see a list of scopes (permissions) that the token can have. If you are new to GitHub, select ALL the boxes.

  6. Step 6: Generate the Token

    Once you've selected the desired permissions, scroll down and click the Generate token button.

    Your token will be shown on the next screen. Copy the token and save it somewhere safe. You won't be able to see it again once you navigate away from this page! You will need this token OFTEN.

  7. Step 7: Secure Your Token

    Keep your token secure! Treat it like a password. If you believe your token has been compromised, immediately revoke it from the "Personal access tokens" page in your GitHub settings.
